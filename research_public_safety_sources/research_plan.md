# Public Safety Integrity Data Source Research Plan

## Main Question

Evaluate whether annual statistics and case-level data can be obtained for sexual assault, fraud/scams, serious injury, and government-official or political-party integrity analysis, including access conditions, API/account requirements, and automation suitability.

## Subtopics

1. Taiwan official open-data crime statistics
   - Expected: annual/monthly crime categories, downloadable formats, API availability, whether accounts or API keys are needed.

2. Judicial Yuan court-judgment data
   - Expected: case-level judgment access, open-data downloads, zip/extraction requirements, legal/data-use constraints, category search feasibility.

3. Political officials and party-affiliation enrichment
   - Expected: possible official rosters, election/candidate data, legislator/councilor/open-government data, and matching limitations.

4. Automation architecture for n8n
   - Expected: whether scraping or official open-data retrieval is better, and which steps need unzip, parsing, scheduled refresh, and data normalization.

## Synthesis

Compare source reliability, access friction, legal stability, and automation complexity. Try fetching one real sample record from an official source and report the observed fields.
